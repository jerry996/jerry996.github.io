1.jQuery简介

jQuery保持了精简和一个较小的尺寸，通过插件可以扩展jQuery

jQuery-源码阅读，pushStack()入栈方法

首先了解下栈：栈（stack）又名堆栈，它是一种运算受限的线性表。其限制是仅允许在表的一端进行插入和删除运算。栈的规则是先进后出。下面看下jQuery中链式方法的使用：
$(‘ul’).children(‘li’).css(“backgroundColor”,”#CCC”);
首先选择页面中ul元素，再寻找其子元素中li，为其设置背景颜色为灰色。这里，ul首先入栈，子元素li后入栈，则css()最后操作的是li元素不是ul，这里是因为jQuery里所有链式操作都调用了jQuery中的工具方法——pushStack()。下面看下这个神奇函数的源码：
pushStack: function( elems ) { 

    // Build a new jQuery matched element set  
    var ret = jQuery.merge( this.constructor(), elems );  

    // Add the old object onto the stack (as a reference)  
    ret.prevObject = this;  
    ret.context = this.context;  

    // Return the newly-formed element set  
    return ret;  
}

可以看到，首先新建一个ret对象，其中连接（merge）了jQuery和传入新对象（集合），然后设置ret对象的上级元素为this，返回ret。然后再看下返回上级的end()在源码中的实现：
end: function() {
return this.prevObject || this.constructor(null);
}
end()利用了pushStack中的prevObject，返回该对象，若该对象为空，返回构造函数。在源码中多次使用这个入栈函数，如map()、slice()、eq()等常用方法中都调用了这个入栈方法。

jQuery是一个自执行的匿名函数，它接收两个参数，——第一个参数是window对象，第二个参数是undefined；
(function ( window, undefined)
{…} //jQuery核心定义

事件处理模型 ：事件冒泡、捕获

dom事件流同时支持两种事件模型，addEventListener第三个参数用来设置事件在捕获(true)阶段执行还是冒泡(false)阶段执行。捕获顺序为先捕获后冒泡，两种事件流都会触及dom中所有的对象，从documment开始到document对象结束

  • 1.事件冒泡：
    子元素事件触发，事件沿着包含关系往上级传递，每一级都可以感知到事件。用event.stopPropagation();终止事件的广播分发，这样事件停留在本节点，不会向外传播。如果给整个document定义了一个点击事件，那么任何元素上的点击事件都会冒泡到document从而触发这个方法。除非之前有一个时间处理函数内部明确终止冒泡，那他就不会一路网上知道document。
    事件冒泡作用：用户鼠标范围特别大，而程序没有跟上，鼠标就不位于那一层之上了。如：onmousemove，此时把事件触发注册在document级别可以让他们及时触发
  • 2.事件捕获：
    结构上（非视觉上）嵌套关系的元素，会存在事件捕获的功能，即同一事件，自父元素捕获至子元素（事件源元素自执行）。（自上向底）addEventListener第三个参数为true
    IE没有捕获事件
  • 3.触发顺序：先捕获，后冒泡。自执行时谁先绑定谁先执行
  • 4.focus，blur，change，submit，reset，select 等事件不冒泡

  • 通过将参数从传入函数，在执行时就可以捕获函数所创建闭包中的变量。闭包就是这样一个函数：它处在一个函数体中，并引用了函数体当前执行上下文中的变量。

    （理解：多个函数互相嵌套，把里面的函数保存到外面函数的外部， 就生成闭包。里面的函数在外部执行的时候，就能调用原来函数所在环境里的变量）垃圾清理机制：把游离的对象清除.
    内存泄露：闭包导致作用域链不释放就导致内存少了。过度的占用系统资源，当成内存泄露，泄露的多剩下的就少了。（沙子从指缝间流逝出去之后手里的就少了）（其实没泄露）

闭包的作用

  • 1.实现公有变量
    eg:函数累加器（内部函数被保存到外部最直接的写法：把单个变量返回，外部用函数去接收这个变量，相当于这个函数被保存到外部）
    a执行完AO被剪断之后，add的AO没有被剪断，所以num可以实现累加

  • 2.可以做缓存（存储结构）
    eg:eater（把两个函数变成一个对象的两个属性返回出去，给到外面一个变量里去，即把这两个函数都保存到外面，就形成了一个共享闭包）
    food起到了一个缓存的作用，只有test通过调用eatFood和pushfood实现对food的一个修改和输出。可以理解为food被person私有化了。
    I am eating apple
    There is nothing
    I am eating banana

  • 3.可以实现封装，属性私有化。
    理解访问级别：默认情况下，对象中所有成员都是公有可访问的。但在JavaScript中可以实现私有属性类似的访问级别效果，要实现私有方法或属性，请使用闭包。

  • 4.防止污染全局变量，模块化开发
    在JavaScript中为了防止命名冲突，我们应该尽量避免使用全局变量和全局函数，可以把要定义的所有内容放入到一个立即执行函数中，相当于给他们外层添加了一个函数作用域。

  • 5.把变量作为参数传入函数时，可以在函数执行时所创建的闭包中捕获变量的值。无论在外部作用域中的值发生了什么变化，闭包将记住函数执行时变量的值。

立即执行函数

立即调用的函数表达式：本质是将一个函数表达式包装在一对圆括号中，迫使JavaScript引擎将function(){}块识别为一个函数表达式，而不是一个函数语句，然后立即调用该函数。

  • 1.执行完之后立即销毁函数本身
  • 2.接收返回值可以定义变量，用立即执行函数初始化数据

闭包的防范

数组里放了10个函数，这十个函数被保存到了外部，这十个函数都和test形成了闭包，都保存了同一个test的劳动成果，里面包含一个变量i，被保存出去之前i变成10了
一个函数和十个函数同时形成闭包，共用一个保存出来的结果
解决办法：把它变成一对一的关系。立即执行函数也可以产生作用域，里面函数被保存到外部拿的是立即执行函数的成果，一共产生10个不同的立即执行函数，每个立即执行函数都和上一次不一样，因为执行完就销毁
